# 蓝海跨境商品竞争指数模型（V4 严谨版）

报告日期：2026-06-24  
适用报告：蓝海跨境商品数据分析报告 V3  
用途：将原先的经验型竞争指数升级为可采集、可复算、可审计的半定量模型。

---

## 1. V4 升级说明

V3 中的竞争指数属于“规则型估算”，适合初筛，但不够严谨。  
V4 将竞争指数升级为：

```text
基于关键词样本 + 平台搜索结果样本 + 价格/评论/广告/同质化特征
计算得到的可复算竞争指数。
```

V4 的核心改进：

| 项目 | V3 | V4 |
|---|---|---|
| 数据来源 | 公开趋势 + 经验判断 | 固定关键词样本 + 平台前 N 条商品样本 |
| 计算方式 | 人工估算 | 公式化评分 |
| 可复算性 | 中低 | 高 |
| 可解释性 | 中 | 高 |
| 适合用途 | 方向初筛 | 品类排序、测款优先级、采购前验证 |
| 局限 | 缺少真实平台样本 | 仍依赖采样质量，不等于平台全量数据 |

---

## 2. 竞争指数总公式

竞争指数满分 100，分数越高代表竞争越激烈。

```text
Competition Index =
0.25 × Supply Density Score
+ 0.20 × Review Concentration Score
+ 0.15 × Ad Pressure Score
+ 0.15 × Price Compression Score
+ 0.15 × Homogeneity Score
+ 0.10 × Brand Lock-in Score
```

### 2.1 权重变化说明

相比 V3，V4 做了两点调整：

1. 将“头部品牌集中度”拆成 Review Concentration 和 Brand Lock-in。
2. 降低广告竞价权重，因为在没有真实 CPC 数据时，广告压力只能通过代理指标估算。

| 特征 | 权重 | 是否必须采集 | 数据类型 |
|---|---:|---|---|
| Supply Density Score，供给密度 | 25% | 必须 | 平台搜索结果 |
| Review Concentration Score，评论集中度 | 20% | 必须 | 商品评论数 |
| Ad Pressure Score，广告压力 | 15% | 建议 | Sponsored 数量 / CPC |
| Price Compression Score，价格压缩 | 15% | 必须 | 商品价格 |
| Homogeneity Score，同质化 | 15% | 必须 | 人工/视觉/文本评分 |
| Brand Lock-in Score，品牌锁定 | 10% | 必须 | 品牌与店铺结构 |

## 3. 数据采集口径

### 3.1 每个品类至少采集 5-10 个关键词

每个候选品类不能只看一个关键词，必须覆盖核心词、长尾词、场景词和高意图词。

示例：Press-on Nails

| 关键词类型 | 示例 |
|---|---|
| 核心词 | press on nails |
| 长尾词 | short press on nails |
| 场景词 | bridal press on nails |
| 风格词 | jade marble press on nails |
| 用户词 | reusable press on nails |

示例：厨房微改造

| 关键词类型 | 示例 |
|---|---|
| 核心词 | peel and stick backsplash |
| 风格词 | peel and stick zellige tile |
| 场景词 | renter friendly kitchen backsplash |
| 套装词 | kitchen refresh kit |
| 材质词 | peel and stick marble countertop |

### 3.2 每个平台采集前 50 条结果

建议采样平台：

| 平台 | 采集条数 | 作用 |
|---|---:|---|
| Amazon | 每关键词前 50 条 | 判断交易竞争 |
| Etsy | 每关键词前 50 条 | 判断设计/手作/礼品化竞争 |
| TikTok Shop | 每关键词前 50 条 | 判断内容电商竞争 |
| Google Shopping | 每关键词前 30-50 条 | 判断全网价格和品牌 |

如果时间有限，最低采样标准：

```text
每个品类 5 个关键词
× 每个平台前 30 条
× 至少 2 个平台
= 每个品类至少 300 条样本
```

### 3.3 单条商品样本字段

每条商品至少记录以下字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| category | 文本 | 是 | 候选品类 |
| keyword | 文本 | 是 | 搜索关键词 |
| platform | 枚举 | 是 | Amazon/Etsy/TikTok/Google Shopping |
| rank | 数字 | 是 | 搜索结果排名 |
| title | 文本 | 是 | 商品标题 |
| brand_or_shop | 文本 | 是 | 品牌或店铺 |
| price_usd | 数字 | 是 | 售价，统一美元 |
| review_count | 数字 | 是 | 评论数，没有填 0 |
| rating | 数字 | 建议 | 星级评分 |
| is_sponsored | 0/1 | 建议 | 是否广告位 |
| is_big_brand | 0/1 | 是 | 是否明显品牌/大店 |
| image_style_group | 文本 | 建议 | 主图相似分组 |
| core_material | 文本 | 建议 | 材质 |
| core_function | 文本 | 建议 | 核心功能 |
| differentiation_note | 文本 | 建议 | 差异化点 |

## 4. 六个子分数的计算方法

### 4.1 Supply Density Score，供给密度分

衡量同类商品供给是否拥挤。

#### 输入特征

| 特征 | 说明 |
|---|---|
| result_count | 平台显示的搜索结果数量 |
| valid_listing_count | 前 N 条中有效相关商品数量 |
| duplicate_rate | 前 N 条中高度重复/同款比例 |

#### 计算公式

```text
Supply Density Raw =
0.50 × Result Count Score
+ 0.30 × Valid Listing Score
+ 0.20 × Duplicate Rate Score
```

归一化方式：

| Result Count | Result Count Score |
|---:|---:|
| < 1,000 | 20 |
| 1,000-5,000 | 40 |
| 5,000-20,000 | 60 |
| 20,000-100,000 | 80 |
| > 100,000 | 100 |

Valid Listing Score：

```text
Valid Listing Score = 前 N 条相关商品数量 / N × 100
```

Duplicate Rate Score：

```text
Duplicate Rate Score = 高度相似商品数量 / N × 100
```

解释：

- 搜索结果越多，竞争越强。
- 前排越多有效商品，说明平台供给成熟。
- 重复同款越多，说明价格竞争风险更高。

---

### 4.2 Review Concentration Score，评论集中度分

衡量前排是否被老链接和大评论商品占据。

#### 输入特征

| 特征 | 说明 |
|---|---|
| median_review_top20 | 前 20 条评论中位数 |
| top10_review_share | 前 10 条评论数占前 50 条评论总数比例 |
| high_review_listing_ratio | 前 50 条中评论数 > 1,000 的商品占比 |

#### 计算公式

```text
Review Concentration =
0.40 × Median Review Score
+ 0.35 × Top10 Review Share Score
+ 0.25 × High Review Listing Score
```

Median Review Score：

| 前 20 条评论中位数 | 分数 |
|---:|---:|
| 0-50 | 20 |
| 51-200 | 40 |
| 201-800 | 60 |
| 801-2,000 | 80 |
| > 2,000 | 100 |

Top10 Review Share Score：

```text
Top10 Review Share Score = min(100, top10_review_share × 120)
```

High Review Listing Score：

```text
High Review Listing Score = high_review_listing_ratio × 100
```

解释：

- 前排评论越高，新品越难进入。
- 评论集中在少数头部链接，说明头部垄断更强。
- 若前 20 条很多是低评论商品，说明仍有新链接机会。

---

### 4.3 Ad Pressure Score，广告压力分

衡量该关键词是否被大量广告竞争。

#### 输入特征

| 特征 | 说明 |
|---|---|
| sponsored_ratio_top20 | 前 20 条广告商品占比 |
| sponsored_above_fold | 首屏广告数量 |
| cpc_proxy | CPC 代理值或工具 CPC |

#### 计算公式

```text
Ad Pressure =
0.45 × Sponsored Ratio Score
+ 0.35 × Above Fold Sponsored Score
+ 0.20 × CPC Score
```

Sponsored Ratio Score：

```text
Sponsored Ratio Score = sponsored_ratio_top20 × 100
```

Above Fold Sponsored Score：

| 首屏广告数量 | 分数 |
|---:|---:|
| 0 | 10 |
| 1-2 | 30 |
| 3-4 | 60 |
| 5-6 | 80 |
| > 6 | 100 |

CPC Score：

| CPC 或代理 CPC | 分数 |
|---:|---:|
| < 0.3 美元 | 20 |
| 0.3-0.8 美元 | 40 |
| 0.8-1.5 美元 | 60 |
| 1.5-3.0 美元 | 80 |
| > 3.0 美元 | 100 |

如果没有 CPC 数据，则 CPC Score 使用 50 作为中性值，并降低模型置信度。

---

### 4.4 Price Compression Score，价格压缩分

衡量价格是否被压到难以盈利。

#### 输入特征

| 特征 | 说明 |
|---|---|
| median_price | 前 50 条价格中位数 |
| low_price_ratio | 低于健康价格带下限的商品占比 |
| price_iqr | 价格四分位距，衡量价格分散度 |
| estimated_landed_cost | 到岸成本估算 |

#### 健康价格带

不同品类健康价格带不同，应先设定目标区间：

| 品类 | 健康价格带 |
|---|---:|
| 美甲/小配饰 | 12.99-29.99 美元 |
| 家居微改造 | 24.99-89.99 美元 |
| 亚麻软装 | 24.99-89.99 美元 |
| 头皮工具 | 9.99-29.99 美元 |
| 香氛周边 | 9.99-39.99 美元 |

#### 计算公式

```text
Price Compression =
0.45 × Low Price Ratio Score
+ 0.35 × Margin Pressure Score
+ 0.20 × Price Narrowness Score
```

Low Price Ratio Score：

```text
Low Price Ratio Score =
低于健康价格带下限的商品数量 / N × 100
```

Margin Pressure Score：

```text
Margin Pressure Score =
max(0, 100 - ((median_price - estimated_landed_cost) / median_price × 100))
```

Price Narrowness Score：

```text
Price Narrowness Score =
100 - min(100, price_iqr / median_price × 200)
```

解释：

- 低价商品占比越高，越容易打价格战。
- 中位价扣除成本后空间越小，利润压力越高。
- 价格分布越窄，说明消费者和卖家都被锁在固定低价区间。

---

### 4.5 Homogeneity Score，同质化分

衡量商品是否高度相似，是否有差异化空间。

同质化很难完全自动化，V4 建议使用“人工评分 + 可观察字段”的混合方式。

#### 输入特征

| 特征 | 说明 |
|---|---|
| visual_similarity | 主图/外观相似度 |
| title_similarity | 标题和卖点相似度 |
| function_similarity | 功能差异程度 |
| material_similarity | 材质差异程度 |
| bundle_variation | 是否有套装/组合差异 |

#### 人工评分标准

每个关键词前 20 条商品，人工打分：

| 分数 | 判断 |
|---:|---|
| 20 | 商品形态差异明显，设计空间大 |
| 40 | 有部分相似，但存在清晰差异 |
| 60 | 多数商品相似，少量差异 |
| 80 | 外观、功能、价格高度相似 |
| 100 | 几乎全是同款/白牌复制 |

#### 计算公式

```text
Homogeneity Score =
0.40 × Visual Similarity
+ 0.25 × Title Similarity
+ 0.20 × Function Similarity
+ 0.15 × Material Similarity
```

如果商品可明显通过套装、定制、尺寸、材质差异化，则可下调 5-15 分。

---

### 4.6 Brand Lock-in Score，品牌锁定分

衡量该品类是否已经被大品牌、强店铺或官方授权锁定。

#### 输入特征

| 特征 | 说明 |
|---|---|
| big_brand_ratio_top20 | 前 20 条大品牌占比 |
| official_store_ratio | 官方店/授权店占比 |
| repeat_seller_ratio | 前 50 条中重复卖家占比 |
| brand_search_dependency | 关键词是否强依赖品牌名 |

#### 计算公式

```text
Brand Lock-in =
0.40 × Big Brand Ratio Score
+ 0.25 × Official Store Score
+ 0.20 × Repeat Seller Score
+ 0.15 × Brand Dependency Score
```

Brand Dependency Score：

| 情况 | 分数 |
|---|---:|
| 完全通用品类词 | 20 |
| 有少量品牌词影响 | 40 |
| 头部品牌明显，但非绝对 | 60 |
| 大量搜索围绕品牌 | 80 |
| 几乎必须靠品牌购买 | 100 |

解释：

- 如果消费者搜索的是“Dyson accessory”“Stanley cup lid”，可能有流量，但品牌依赖和侵权风险更高。
- 如果前排都是通用品类卖家，品牌锁定分较低。

## 5. 跨平台加权

不同平台对不同商品的重要性不同。V4 不建议简单平均。

### 5.1 平台权重建议

| 商品类型 | Amazon | Etsy | TikTok Shop | Google Shopping |
|---|---:|---:|---:|---:|
| Press-on Nails / 美甲贴 | 25% | 25% | 40% | 10% |
| 厨房微改造 | 45% | 10% | 30% | 15% |
| 洗水亚麻软装 | 25% | 45% | 10% | 20% |
| 头皮工具 | 45% | 5% | 40% | 10% |
| 胸针/衣领针 | 25% | 50% | 15% | 10% |
| Walking Pad 周边 | 60% | 0% | 15% | 25% |

### 5.2 跨平台公式

```text
Final Competition Index =
Amazon Score × Amazon Weight
+ Etsy Score × Etsy Weight
+ TikTok Score × TikTok Weight
+ Google Shopping Score × Google Weight
```

如果某个平台无法采集：

1. 不建议直接填 0。
2. 使用剩余平台权重重新归一化。
3. 降低置信度等级。

## 6. 置信度等级

竞争指数必须搭配置信度使用。

| 置信度 | 采样要求 | 可用程度 |
|---|---|---|
| A | 每品类 ≥ 8 个关键词，≥ 3 个平台，≥ 800 条样本，有 CPC 或广告代理数据 | 可用于采购前严肃决策 |
| B | 每品类 ≥ 5 个关键词，≥ 2 个平台，≥ 300 条样本，有价格和评论数据 | 可用于测款优先级 |
| C | 每品类 < 300 条样本，或缺广告/评论/价格核心字段 | 仅用于初筛 |
| D | 主要依赖报道和经验判断 | 不建议用于采购 |

V3 报告中的竞争指数大多应视为 C-D 级。  
V4 完整采集后可提升到 B 级，若配合付费工具和平台后台可提升到 A 级。

## 9. Python 计算伪代码

```python
def weighted_score(parts):
    return sum(score * weight for score, weight in parts)

competition_index = weighted_score([
    (supply_density_score, 0.25),
    (review_concentration_score, 0.20),
    (ad_pressure_score, 0.15),
    (price_compression_score, 0.15),
    (homogeneity_score, 0.15),
    (brand_lockin_score, 0.10),
])
```

跨平台：

```python
final_score = (
    amazon_score * amazon_weight
    + etsy_score * etsy_weight
    + tiktok_score * tiktok_weight
    + google_score * google_weight
)
```

## 11. V4 结论

V4 竞争指数不是简单主观评分，而是一个可以落地采集和复算的半定量模型。

它最关键的 6 个特征是：

1. 供给密度：市场上同类商品多不多。
2. 评论集中度：前排是否被老链接占住。
3. 广告压力：关键词是否需要高成本买流量。
4. 价格压缩：价格是否已经被打到难盈利。
5. 同质化程度：商品是否全都长得一样。
6. 品牌锁定：消费者是否被大品牌或授权体系锁住。

使用建议：

- V3 用来确定候选方向。
- V4 用来决定哪些方向进入真实打样和采购。
- 真正下单前，至少做到 B 级置信度：每品类 5 个关键词、2 个平台、300 条以上样本。
